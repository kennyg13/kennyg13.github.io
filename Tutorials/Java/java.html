Lesson 1 - Declaring Variables and Methods

Now to actually learning to program, there are three variables in this Java class, these three are the basics, which are basically objects that hold certain information. Just know to close a variable you need to end it with a ; So the int a = 1; is holding the whole number 1 and it's type is integer(Whole Number).

String b = "Kenneth" is a String variable which holds word(s), declared by using its type String naming the variable whatever you like as long as its not a number, in this case b and hold the string Kenneth be sure though to have the quotes around the word you want your variable to hold or else it won't work and you'll get an error.

boolean c = false; is a boolean variable which holds either true or false. This variable is called c and holds false

Now I'm going to show you how to create a method that will help things go a lot quicker. A method is something that holds functionallity(Code that does something), you can then use that method to manipulate variables or do certain things now the method I created is public static void print(Object a) and the opening and closing brackets contain the functionallity that allows me do something, without them you'll get an error.

Now the part where it says print(Object a) means that when I tell the computer to do whatever is inside the method print I can give it a variable to manipulate. That variable is type Object(Any variable) and is called a. How do you tell the program to run the method print? By calling it and all you have to do to call a method is state it's name and pass in the variable it asks for like on line 6 in the Java.java class.

I am calling print(Object a) and passing in the String Kenneth what this allows me to do is use the variable in the method print, where as I couldn't before. Now that I have passed the variable b into print it is no longer known as b but as a because that is the name of the variable in the parameter of the method print(Object a). When I manipulate the variable a it doesn't change the String b because right now there is now way for me to save it back to the main method.
